## 코딩 스타일 및 컨벤션
- PEP 8 스타일 가이드를 준수합니다.
- 모든 함수와 클래스에는 타입 힌트(`Type Hinting`)를 명시
- 함수 및 클래스에는 Docstring을 필수적으로 작성
- 함수명은 동사로 시작합니다.
- 변수명: `snake_case`
- 클래스명: `PascalCase`
- 상수: `UPPER_SNAKE_CASE`
- Python이 아닌 다른 언어에 대해서는 각 언어에서 공식적으로 규정하는 코딩 스타일 및 컨벤션을 따르도록 한다.

## README.md 문서화 스타일
- 프로젝트 문서화 시 아래의 표준 구조를 준수하며, 객관적이고 기술적인 문체를 사용합니다.

1. Introduction (개요): 프로젝트의 핵심 목적과 해결하고자 하는 기술적 문제를 1~2문장으로 요약합니다.

2. Key Features (주요 기능): 시스템의 핵심 기능을 기술적 관점에서 나열합니다. (예: "비동기 처리를 통한 데이터 처리 속도 최적화", "PostgreSQL 기반의 관계형 데이터 모델링")

3. Project Structure (프로젝트 구조): 디렉토리 구조를 트리 형태로 제시하고, 각 주요 모듈 및 패키지의 역할을 간결하게 설명합니다.

4. System Workflow (실행 흐름): 어플리케이션의 초기화부터 종료까지의 생명주기(Lifecycle) 또는 주요 데이터 흐름을 단계별로 기술합니다.

5. Getting Started (시작하기):
    - Prerequisites: 실행에 필요한 런타임 환경 및 의존성 도구 명시.
    - Installation & Setup: 라이브러리 설치 및 환경 변수(.env) 설정 과정을 기술합니다.
    - Usage: 표준 실행 명령어 및 주요 파라미터 사용법을 안내합니다.

6. Configuration (설정): 시스템 동작에 영향을 주는 주요 설정값이나 상수를 표(Table) 형식을 활용하여 설명합니다.
